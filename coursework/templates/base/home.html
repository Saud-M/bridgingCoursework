Building a website with Django Framework

This is my first time working with Django framework, also this is the first website i’ve ever built. Django surprisingly makes the process so easy. When I look for anything that comes to my mind I find how to make it using the framework and how simple it is. So my website is structured as follows: There is a base page that includes a navigation bar and footer, and any other page extends my base page. After setting up the pages I want, I went through many templates on the internet. And Bootstrap was really helpful where I used a template for the home, contact, and cv pages.There are two applications, one for contact where a visitor can send a message and I’ll receive it in the admin page. The second one is the CV application where I can add and remove almost anything in my CV. Thanks to Django Framework, building this website wasn’t difficult and was a time saver.

Test-Driven-Development

The second part of this coursework was about Test-Driven-Development (TDD), and I didn’t hear about it before. I did some research and went through the provided book to know what is the procedure exactly and how should I do it. After finishing the first six chapters I thought I had enough knowledge to start. My problem in testing is sometimes there are trivial tests and I’m not sure if I should write them, however, I make it like a scenario to test the major things in my website. I make a test for the welcome page to be sure it’ll transfer within less than 7 seconds. Then I did a few tests on the Contact App to be sure that the message will be sent if the details are correct. And the last test was for CV App, so after I make my application, the test will show if the courses for example will appear in the CV page, it didn’t at first but then everything is fixed and working fine. All of the testing can be found in functional_tests.py. Testing in general might be boring, but when I get used to it I feel like it makes the programming and catching silly mistakes much easier.
